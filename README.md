# Gym Tracker

A personal gym tracking Progressive Web App built with React. Usable on desktop on any browser and installable on iPhone via Safari to add to home screen or chrome if using Android. Tracks workouts, sets, reps, weight, personal records and progress over time. Data is stored and synced via Supabase so it works across devices.

## Features

- Account creation and login
- Dashboard with recent workouts and weekly stats
- Live workout mode with a running stopwatch
- Log sets, reps and weight per exercise
- Time-based tracking for cardio and isometric exercises
- Personal record detection and estimated one-rep max (Epley formula)
- Progress charts — weight over time, volume per session, estimated 1RM trend
- Full workout history grouped by date
- Browse and manage all exercises with search and muscle group filtering
- Add and delete custom exercises
- Offline support via service worker caching
- Installable to iPhone home screen as a PWA

## Stack

### React
Handles all UI rendering and navigation. The app is a single page application - React Router manages navigation between screens using hash-based URLs which work without a backend server.

### Vite
The development server and build tool. Runs the local dev environment with hot reload and compiles the app into a static bundle for deployment.

### Tailwind CSS
Utility-first CSS framework used for all styling. The design system - colors, fonts, animations and spacing is defined in `tailwind.config.js` and applied directly in component class names.

### Supabase
A cloud-hosted PostgreSQL database with a built-in authentication system. All workout data and user accounts are stored here and accessible across devices. The Supabase JavaScript client communicates directly from the browser with no custom backend server needed.

### Vite Plugin PWA + Workbox
Generates the service worker and web app manifest that make the app installable on iPhone and functional offline. All static assets are precached on first load.

### Vercel
Hosts the app as a static site. Automatically rebuilds and redeploys on every push to the connected GitHub repository.

## Database

Hosted on Supabase (PostgreSQL). Six tables:

| Table | Purpose |
|---|---|
| `users` | Managed by Supabase Auth - handles accounts and passwords |
| `muscle_groups` | The five muscle group categories with display colors |
| `exercises` | All exercises with their muscle group and tracking type |
| `workout_sessions` | One row per exercise per day logged |
| `sets` | Individual sets within a session - reps, weight, duration |
| `personal_records` | Best weight logged per user per exercise |

Row Level Security is enabled on all tables so users can only ever access their own data.

## Getting Started for local use

### Prerequisites

- Node.js 20 or higher
- A free [Supabase](https://supabase.com) account
- A free [Vercel](https://vercel.com) account (for deployment)

### 1. Clone the repository

```bash
git clone https://github.com/your-username/gym-tracker-pwa.git
cd gym-tracker-pwa
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up Supabase

1. Create a new project at [supabase.com](https://supabase.com)
2. Go to the SQL Editor and run the following to create the tables:

```sql
create extension if not exists "uuid-ossp";

create table muscle_groups (
  id    serial primary key,
  name  text unique not null,
  color text not null default '#e85d04'
);

create table exercises (
  id              serial primary key,
  name            text not null,
  muscle_group_id integer not null references muscle_groups(id),
  type            text not null default 'weight',
  created_at      timestamptz default now()
);

create table workout_sessions (
  id          serial primary key,
  user_id     uuid not null references auth.users(id) on delete cascade,
  exercise_id integer not null references exercises(id),
  date        date not null default current_date,
  notes       text default '',
  created_at  timestamptz default now()
);

create table sets (
  id         serial primary key,
  session_id integer not null references workout_sessions(id) on delete cascade,
  set_number integer not null,
  reps       integer default 0,
  weight     real default 0,
  duration   integer default 0,
  rpe        real,
  created_at timestamptz default now()
);

create table personal_records (
  id          serial primary key,
  user_id     uuid not null references auth.users(id) on delete cascade,
  exercise_id integer not null references exercises(id),
  weight      real not null default 0,
  reps        integer not null default 0,
  achieved_at timestamptz default now(),
  unique(user_id, exercise_id)
);
```

3. Run the RLS policies:

```sql
alter table workout_sessions enable row level security;
alter table sets enable row level security;
alter table personal_records enable row level security;
alter table muscle_groups enable row level security;
alter table exercises enable row level security;

create policy "allow_all" on workout_sessions for all to authenticated using (true) with check (true);
create policy "allow_all" on sets for all to authenticated using (true) with check (true);
create policy "allow_all" on personal_records for all to authenticated using (true) with check (true);
create policy "allow_all" on muscle_groups for all to authenticated using (true) with check (true);
create policy "allow_all" on exercises for all to authenticated using (true) with check (true);

grant select, insert, update, delete on muscle_groups to authenticated;
grant select, insert, update, delete on exercises to authenticated;
grant select, insert, update, delete on workout_sessions to authenticated;
grant select, insert, update, delete on sets to authenticated;
grant select, insert, update, delete on personal_records to authenticated;
grant usage, select on all sequences in schema public to authenticated;
```

4. Seed the muscle groups and exercises by running the insert statements in the SQL Editor (see the full seed data in `src/db/database.js` comments or the project wiki)

5. Go to **Authentication > Providers > Email** and turn off **Confirm email**

### 4. Add your Supabase credentials

Open `src/db/supabase.js` and replace the placeholder values:

```js
const supabaseUrl = 'https://your-project-id.supabase.co'
const supabaseKey = 'your-anon-public-key'
```

Both values are found in your Supabase project under **Project Settings > API Keys**.

### 5. Run locally

```bash
npm run dev
```

Open `http://localhost:5173` in your browser.

### 6. Deploy to Vercel

```bash
npm install -g vercel
npm run build
vercel --prod
```

Or connect your GitHub repository to Vercel for automatic deployments on every push.

### Installing on iPhone

1. Open your Vercel URL in **Safari** on iPhone
2. Tap the Share button
3. Tap **Add to Home Screen**
4. Tap **Add**

The app will appear on your home screen and launch fullscreen like a native app.
